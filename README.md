# AI Coding 项目开发经验指南

> 一套经过实战验证的 AI 辅助编程最佳实践，专为提升代码质量、开发效率和项目可维护性而设计。

## 📖 项目简介

本项目整理了在使用 Kiro 等 AI Coding 工具进行软件开发过程中积累的核心经验和最佳实践。通过系统化的原则指导，帮助开发者和 AI 工具协作产出更高质量的代码和架构设计。

## 🎯 核心价值

- **提升代码质量**: 通过 SOLID 原则和设计模式确保代码的可维护性
- **优化开发效率**: 基于简洁性和实用性原则，避免过度工程化
- **增强系统可观测性**: 规范化的日志记录和监控策略
- **改善协作体验**: 高级沟通策略提升人机协作效果

## 📚 指南文档

### 🏗️ [面向对象设计 SOLID 原则实践指南](./solid_principles.md)
专为 AI 代码生成优化的面向对象和组件化设计核心准则，确保代码具备：
- **单一职责**: 每个单元只对一个功能负责
- **开放封闭**: 对扩展开放，对修改封闭
- **里氏替换**: 子类必须能够替换其父类
- **接口隔离**: 客户端不应依赖不使用的方法
- **依赖倒置**: 高层模块依赖抽象而非具体实现

### 🎨 [软件开发核心设计原则](./software-design-principles.md)
涵盖奥卡姆剃刀、KISS、YAGNI、DRY 等通用软件设计最佳实践：
- **奥卡姆剃刀**: 选择最简单、最直接的方案
- **KISS 原则**: 保持简单，让代码易于理解和维护
- **YAGNI 原则**: 只实现当前明确要求的功能
- **DRY 原则**: 避免任何形式的逻辑重复
- **高内聚低耦合**: 优化模块间的组织和依赖关系

### 📊 [结构化日志记录规范](./structured-logging-guide.md)
AI 生成日志代码的专业标准，包含性能优化、安全合规和多语言实现模板：
- **7条核心铁律**: 从结构化格式到指标分离的完整规范
- **性能优化策略**: 避免常见性能陷阱，优化高频场景
- **安全合规要求**: 敏感信息处理和数据脱敏规范
- **多语言代码模板**: Python、Go、Node.js 等主流语言示例

### 🤝 [人机协作沟通策略](./ai-collaboration-guide.md)
提升开发者与 AI 工具协作效率的高级沟通方法和最佳实践：
- **需求挖掘与重塑**: 从表面需求探究真实业务目标
- **上下文化解决方案**: 交付包含决策思路和风险分析的完整方案
- **主动规划与赋能**: 从被动执行转向主动技术伙伴

## 💡 使用建议

### 对于开发者
- **渐进式应用**: 从一个原则开始，逐步融入到日常开发中
- **团队共识**: 与团队成员分享这些原则，建立统一的代码标准
- **持续改进**: 根据项目实际情况调整和优化这些原则

### 对于 AI 工具
- **系统性指导**: 将这些文档作为 AI 工具的系统提示或指导原则
- **上下文注入**: 在具体任务中引用相关原则，确保输出符合标准
- **质量检查**: 使用文档中的检查清单验证生成代码的质量
